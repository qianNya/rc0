## 1. 体验原则

### 1.1 五条北极星原则

1. **Content is the world**：内容是空间本体，UI 是浮在内容上的光学层。用户第一眼看到的必须是剧照/分镜/角色/生成图，而不是导航壳。
2. **界面退后，创作向前**：工具在需要时浮现（Emerges），用完即消融（Dissolves）。编辑器中 Storyboard 是内容层，Timeline/参数面板是工具层，工具层不得视觉压制内容层。
3. **一切皆可回溯到剧本**：任何图片、角色、场景、预设都能导航回其所属的 `screenplay/scene/frame`，用户永远不会"迷失在资产里"。
4. **液态连续性**：页面、Tab、面板之间的切换是液体般的融合过渡，没有硬切换、没有白屏闪烁。
5. **离线优先，云端透明**：本地草稿永远可编辑；同步/上传状态可见但不打断创作流。

### 1.2 体验反模式（出现即重构）

- 后台管理式的密集表格/表单页。
- 需要用户理解"本地图 vs 远端图"技术差异的界面。
- 弹窗套弹窗、超过 2 层的模态堆叠。
- 保存/发布前不可见的破坏性覆盖。

## 2. 信息架构与导航

### 2.1 五大心智入口（shell 主导航）

| 入口 | 路由 | 用户心智 |
|---|---|---|
| 模板 | `/discovery` | "浏览可 Fork 的剧本模板"（单页模板市场） |
| 场景 | `/scenes` | "场景库与拍摄空间" |
| 资产 | `/assets` | "角色 / 设备 / 可复用素材枢纽" |
| 我的 | `/profile` | "作品、身份与设置" |
| 创作（独立） | `/studio` | "继续我的剧本 / 新建创作"（不计入消费 Tab） |
| 社区（deprecated） | `/community` → `/discovery?section=template` | 旧链接重定向 |

双端 L1 标签与顺序一致（模板 / 场景 / 资产 / 我的）。App：浮动底栏 1–5 槽（`ShellNavConfigStore`）+ 独立创作胶囊。PC（≥840）：侧栏 L1 + 可展开 L2 + 底部创作 CTA。详见 [SHELL_NAV_PRD.md](../refactor/SHELL_NAV_PRD.md)。

### 2.2 页面层级（L1–L5）

| 层级 | 类型 | 示例 | 交互约束 |
|---|---|---|---|
| L1 | Shell 主入口 | `/discovery` `/studio` | 浮动导航常驻 |
| L2 | 领域列表/Hub | `/script` `/character` `/scenes` `/preset` | 支持筛选/搜索，浮动 Tab |
| L3 | 详情/阅读 | `/script/:id` `/character/:id` | Hero 沉浸式，只读+轻操作 |
| L4 | Studio 工作页 | `/studio/edit/.../frame/...` | 全屏编辑态，导航退场 |
| L5 | Sheet/Picker | 角色选择、预设选择 | GlassSheet，单层，用完即散 |

规则：**阅读态与编辑态严格分离**。详情页（L3）不承载编辑器草稿状态；编辑页（L4）不作为普通浏览入口。跳转编辑必须显式（"在 Studio 中编辑"）。

### 2.3 导航行为

- Tab 内页面走 shell branch（保留各 tab 滚动位置，`StatefulShellRoute.indexedStack`）。
- 沉浸编辑、详情、登录、资产 CRUD 走 root route（覆盖 shell，返回后恢复原位）。
- 路径可读：列表用复数（`/scenes`）、详情用 `/:id`、编辑用 `/edit`、AI 入口用 `/ai`。
- 需要登录的行为（发布/点赞/关注/上传）在**触发时**引导登录，而不是路由级拦截浏览。

## 3. 核心旅程体验标准

### 3.1 发现 → 阅读

- 首屏 800ms 内出现内容骨架；图片渐进加载（缩略 → 展示图）。
- Feed 卡片以图为主体，元数据（作者/点赞数）弱化为 caption 层级。
- 详情页 Hero 图全屏进入，信息玻璃卡浮于其上；剧本结构以 Act→Scene→Frame 树呈现，可逐层展开阅读。

### 3.2 创作（Studio 主链路）

- 进入 Studio 直接看到"最近编辑"，一键继续；新建剧本 ≤ 2 步到达编辑器。
- 编辑器自动保存本地草稿（无显式"保存"焦虑）；同步状态用轻量指示器（图标 + caption），不用弹窗。
- Frame 编辑器：画面预览是主体；动作/打光/摄影/AI 参数以浮动面板或 GlassSheet 承载，收起后完全让位于画面。
- 绑定角色/场景/预设走 Picker Sheet（L5），选择后立即回到编辑上下文，不发生页面跳转。

### 3.3 AI 生成

- 生成是**异步任务**：提交后立即返回可继续编辑，任务进度在通知/任务入口可查。
- 生成结果必须回写到发起上下文（frame/角色/场景/图库），并展示"应用到分镜"的直接操作。
- 失败要给出可行动的错误（余额/参数/网络），支持一键重试；禁止裸错误码。

### 3.4 沉淀与分享

- 图库中每张图可追溯来源（剧本/分镜/生成任务），点击即回跳。
- 发布前有可见性选择（公开/私密）与封面确认；发布过程用 `GlassProgressSheet` 展示可中断的进度。

## 4. 状态与反馈

### 4.1 三态强制

所有列表/数据页必须实现三态，缺一不可：

| 状态 | 组件 | 要求 |
|---|---|---|
| 加载中 | 骨架屏/占位渐变 | 用 `explorePlaceholderStart/End` 渐变占位，不用转圈遮罩 |
| 空 | `GlassEmptyState` | 一句引导 + 一个行动按钮（"去创建"），不是冷冰冰的"暂无数据" |
| 错误 | 统一错误卡 + 重试 | 文案经 `apiErrorMessage()` 转换，人话化 |

### 4.2 反馈层级

| 反馈类型 | 载体 | 时长 |
|---|---|---|
| 操作确认（已收藏/已复制） | 全局统一 SnackBar 样式 | 2s 自动消失 |
| 需要决策（删除/覆盖） | `GlassDialog` | 用户显式关闭 |
| 长任务（发布/导出/生成） | `GlassProgressSheet` | 可取消，含进度 |
| 401 会话过期 | 统一 SnackBar + "去登录"动作 | 全局去重，不重复弹 |

禁止业务页面覆写私有 SnackBar/Toast 样式。

### 4.3 触觉与微反馈

- 可点击玻璃元素按压时轻微缩放（`emphasized` 曲线，150ms）。
- 关键成功动作（发布成功、生成完成）配轻触觉反馈（移动端）。
- 滚动时导航玻璃对经过的内容实时透镜响应，强化"玻璃浮在内容上"的空间感。

## 5. 输入与操作

- 表单用 `GlassTextField`（无下划线）；键盘弹出时表单自动避让。
- 破坏性操作（删除剧本/分镜）必须 `GlassDialog` 二次确认，且明确说明影响范围（"将同时删除 12 个分镜"）。
- 多选批量操作（剧本管理）用浮动选择条（`ScreenplaySelectionBar` 模式）：进入选择态时导航消融、选择条浮现。
- 拖拽排序（分镜/场次）提供实时预览与 `shadowDrag` 抬升反馈。

---

## 6. 可访问性与包容性

- 玻璃上文字对比度：明暗两套 token 均需满足 WCAG AA（4.5:1 正文 / 3:1 大字）。
- 触控目标 ≥ 44×44pt；底部导航 tab slot 56pt。
- 尊重系统"减弱动态效果"设置：液态动效降级为交叉淡化，不降级功能。
- 图片内容提供语义标签（生成图带 prompt 摘要作为无障碍描述）。

---

## 7. 性能体验红线

- 页面切换动画掉帧 = 缺陷：同屏 blur 层 ≤ 2，玻璃复用共享组件。
- 大列表分页/懒加载；图片一律走 `Rc0Image`（缓存 + WebP），禁止裸 `Image.network`。
- 状态变化局部化（`ListenableBuilder`/provider 精确订阅），避免整页 rebuild。
- 冷启动到首屏内容 ≤ 2s（中端设备）；编辑器打开草稿 ≤ 1s。

---

## 8. 响应式与多端

- compact（<600）：单列，底部浮动导航。
- medium（≥840）：侧栏壳切换，内容双列。
- expanded（≥1024）：桌面三栏（导航/内容/检查器），编辑器左中右布局。
- 同一业务页面在各端保持同一心智模型，只改布局不改流程。

---

## 9. 验收清单（体验走查）

1. 新用户 3 分钟内能完成"看到灵感 → 建剧本 → 生成第一张分镜图"吗？
2. 任何页面能一眼回答"我在哪、能做什么、怎么回去"吗？
3. 断网状态下创作是否不受阻？恢复后是否自动同步？
4. 所有等待是否可见、可取消、可重试？
5. 阅读态与编辑态是否零混淆？
6. 明暗模式、三种断点下走查主链路是否一致？
